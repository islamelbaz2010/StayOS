# EXPERIENCE_RULES.md

As Chief Customer Experience Officer, I am establishing these non-negotiable performance thresholds to ensure the StayOS ecosystem remains human-centric, frictionless, and reliable. These rules are absolute.

## 1. Digital Interaction Thresholds

| Metric | Rule |
| :--- | :--- |
| **Max Clicks (Booking)** | 3 clicks from Search to Confirmation |
| **Max Screens (Booking Flow)** | 3 screens |
| **Max Payment Steps** | 2 steps (Summary & Authorize) |
| **Max Registration Time** | 60 seconds (Total) |
| **Max Search Time** | 2 seconds (to display results) |

## 2. Communication & Response Latency

| Metric | Rule |
| :--- | :--- |
| **Max Initial Response Time (WhatsApp/Chat)** | 30 seconds |
| **Max Waiting Time (Phone Call)** | 20 seconds |
| **Max Push Notification Latency** | 5 seconds (from event trigger) |
| **Max Email Confirmation Delivery** | 60 seconds (post-action) |

## 3. Operations & Resolution SLA

| Metric | Rule |
| :--- | :--- |
| **Max Support Resolution Time** | 15 minutes (Initial contact to solution) |
| **Max Refund Processing Time** | 24 hours (Initiated to funds released) |
| **Max Issue Reporting Time** | 10 seconds (User action to recorded log) |
| **Max Check-in Access Time** | 0 seconds (Automated "walk-up" access) |
| **Max Cleaner Schedule Update Delay** | 1 minute (Change to notification) |

## 4. Operational Compliance Standards

* **Automation Threshold:** 100% of standard guest communications (check-in/check-out) must be automated via pre-scheduled, dynamic triggers.
* **Availability:** 24/7/365 active monitoring. System "Down Time" tolerance is 0 minutes per month.
* **Data Entry:** Manual entry for any actor is capped at 3 fields per interaction.
* **Verification:** Identity verification must be processed in under 2 minutes using passive background checks.

---
*Failure to meet any of these metrics constitutes a violation of the StayOS Experience Charter.*
