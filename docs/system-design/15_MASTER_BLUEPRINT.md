## 1. What We Are Building

**StayOS** is an AI-powered, two-sided accommodation marketplace for MENA — Egypt first, GCC corridor as the business. This is not a computer OS. The platform connects verified property owners (supply) with Arabic-speaking Egyptian and GCC travelers (demand).

**Core differentiators**: Arabic-first UX · Egyptian payment rails (Fawry, Meeza, Vodafone Cash) · Trust infrastructure (mandatory KYC, escrow, verified listings) · WhatsApp-native communication.

See [MASTER_CONTEXT.md](../../MASTER_CONTEXT.md) and [DECISION_LOG.md](../../DECISION_LOG.md).

## 2. Architecture in One Diagram

```
Internet
    ↓
CloudFront CDN (edge cache: photos, static assets)
    ↓
Vercel (Next.js 14 SSR — Arabic RTL, SEO-optimized)
    ↓
AWS ALB (HTTPS/TLS 1.3 termination)
    ↓
FastAPI (ASGI, uvicorn) — Modular Monolith (Phase 1)
    ├── AuthGate      (FC-01) → Firebase + Twilio + Textract + Rekognition
    ├── PMS Core      (FC-02, FC-04) → PostGIS + Google Maps + S3 (photos)
    ├── Reservation   (FC-03) → Paymob + Stripe
    ├── OpsManager    (FC-05) → S3 (ops photos) + WhatsApp
    └── FinancialEngine (FC-06) → Paymob disbursement + AWS SES
    ↓
PostgreSQL 16 + PostGIS 3  |  Redis 7
    ↓
Celery Workers (async) + Celery Beat (scheduled)
    → Notifications (WhatsApp + Email + SMS)
    → Escrow release (T+24h)
    → Payout batch (daily)
    → Outbox event dispatch
```

## 6. API (Summary)

- **Style**: REST + OpenAPI 3.0 (`/api/v1/...`)
- **Auth**: Firebase JWT (`Authorization: Bearer <token>`)
- **Pagination**: Cursor-based
- **Errors**: RFC 7807 Problem Details with `trace_id`
- **Rate limiting**: Redis sliding window (3 OTP/15min, 10 checkout/min, 200 req/min authenticated)
- **~35 endpoints** across 7 resource groups

Full spec: [04_API_SPECIFICATION.md](04_API_SPECIFICATION.md)

## 7. Database (Summary)

- **7 schemas**: auth, pms, reservation, finance, ops, notify, outbox
- **~15 tables** with explicit constraints
- **Key safety features**:
  - GIST exclusion constraint on `calendar_rules` prevents double-booking at DB level
  - Ledger entries are append-only (no UPDATE/DELETE)
  - Escrow balance cannot go negative (CHECK constraint)
  - PostGIS spatial index for < 50ms geo queries at 100K+ listings

Full schema: [05_DATABASE_DESIGN.md](05_DATABASE_DESIGN.md)

## 8. Events (Summary)

**17 domain events** across 5 domains. All use the Transactional Outbox pattern. All are idempotent with Redis NX keys.

Critical events with HIGH priority retry (P0 on failure):
- `booking.payment_confirmed` → starts escrow timer
- `booking.checked_out` → creates turnover ticket (within 5 min)
- `ops.ticket_created` → assigns field staff
- `finance.payout_dispatched` → notifies host

Full catalog: [06_EVENT_CATALOG.md](06_EVENT_CATALOG.md)

## 12. Security (Summary)

- **OWASP Top 10** addressed across all categories
- **No PII in logs, task queues, or analytics events**
- **KYC documents**: S3 with SSE-KMS (CMK), access-controlled, 7-year retention
- **Secrets**: AWS Secrets Manager, rotated every 90 days
- **Webhooks**: HMAC-SHA512 (Paymob), Stripe-Signature header verification
- **Threats addressed**: Fraudulent hosts, payment fraud, account takeover, data scraping, insider threats
- **CI security**: bandit, safety, trufflehog on every PR

Full model: [10_SECURITY_MODEL.md](10_SECURITY_MODEL.md)
